יצירת דפים וראוטים בנקסט :
------------------------------
באמצעות תיקיות. למשל לראוט : 'about /' -- ניצור תיקייה בשם about ובתוכה קובץ בשם page.js/page.jsx 
ואם עובדים עם טייפסקריפט אז page.ts/page.tsx 
ככה נקסט ידע שמדבור בדף ולא בקומפוננטה 

כדי ליצור את דף הבית מכיון שהראוט שלו הוא '/'
אז בעצם הוא נמצא בתיקיה הראשית -- app לכן פשוט נעשה בה קובץ page.jsx שייצג את דף הבית 

תפריט נפתח :
-------------
אם בדף אודות יש תפריטי משנה -- צוות החברה, קריירה אצלינו 
עושים תיקיות משנה בתוך תיקית 
about ובהם קובץ 
page.js/jsx

ככה :

about --> team --> page.jsx
      --> career --> page.jsx

שמות דפים דינאמיים :
----------------------
אם שמות הדפים מגיעים למשל מהדטא בייס 
למשל בבלוג, שם הפוסט הוא או 
ID 
או סלאג 
איך נדע את שם התיקייה [ הראוט ?]

מכניסים את הדבר הדינאמי בשם התקייה בסוגריים מרובעים 

blog --> [slug folder] --> page.jsx

קבוצת דפים :
-------------
נניח שאנחנו רוצים שקבוצת דפים יהיו יחד מצד אחד 
ומצד שני שזה לא ישפיע על הראוט 

הוא הראה דוגמא עם דפי המשתמש :
login, register, forgot-password 

אם נעשה שינוי באחד מהם, זה יכול להשפיע על השאר . ואם זה פרוייקט גדול, לא נרצה לחפש אותם 

אחד הדרכים היא פשוט ליצור תיקייה משותפת למשל נקרא לה auth 
ולהכניס לתוך סוגריים 
זה לא משפיע על הראוט 

למשל לדף הלוגין :
http://localhost:3000/login

ולא 
http://localhost:3000/auth/login

האדר ופוטר :
--------------
האדר ופוטר אני רוצה שיראו בכל הדפים 

לשם כך משתמשים בקובץ:
layout.js

שם ה 
<body className={inter.className}>
    {children}
</body>

{children} --- זה תוכן הדפים ומה שמעליו זה ההאדר ויופיע בכל הדפים 
ומתחתיו זה הפוטר 

הוא הראה שהלייאוט 
לא חייב להיות דווקא עבור האדר ופוטר 

הוא למשל יצר בבלוג בתוך תיקיית ה 
slug 
layout.jsx

שם הוא קיבל לסוגריים את ה 
{children}
ויצר תוכן מעל שיופיע בכל עמודי הבלוג מעל התוכן של הפוסט 

יצירת תיקית קומפוננטות 
--------------------------
הוא אמר שבמדריכים של נקסט באתר שלהם 
הם ממליצים לשים את כל התיקיות כולל תיקיית קומפוננטות בתיקיית 
app
ללא תיקיית src 
מעל 

הוא הסביר שעדיף ליצור תיקית SRC 
ובתוכה 
app, components, lib 
אחרת מאבדים ידיים ורגליים ולא מוצאים דברים 

יצירת התפריט -- navbar 
-------------------------
יצרנו קומפוננטת נב באר 
בשונה מדפים שם היהי תיקיה עם שם הראוט ובתוכה קובץ בשם page.jsx 
זהה לכל התקיות 
פה זה קומפוננטה ולא דף ולכן נקרא לה 
Navbar.jsx 

כמו כן, יש לשים לב ששמנו את הקומפוננטה בקובץ ה 
layout.jsx

loading.jsx
-------------
ניתן בתיקית app
להוסיף קובץ שבו יופיע תוכן כאשר אנו טוענים את הדף 
לקובץ הזה קוראים 
loading.jsx

עיצוב דף 404 :

import Link from 'next/link'

  return (
    <div>
        <h2>Page not found ...</h2>
        <Link href="/">Back to Homepae</Link>

  )
}

export default notFount

קונפיליקטים בעת שימוש בקלאס זהה:
-----------------------------------
באתר מרובה קבצים וקוד,
אם נשתמש באותו קלאס, למשל :
className="container"

כמה םעמים, זה יכול ליצור קונפליקטים ובעיות עיצוב 

לשם כך, יש בנקסט פתרון נחמד מאד . 

א. קוראים לקובץ :
navbar.module.css 
או 
footer.module.css 
וכעת ניתן להטמיע שם קלאס דינאמי [ לא לשכוח לייבא את ה CSS ]

כיצד מטמיעים שם קלאס דינאמי ?

במקום לרשום :
className="container"

נרשום 

className={styles.container}

ועת יבוא קובץ ה CSS 
נרשום :

import styles from "./navbar.module.css"

וכעת בגלל ששם הקלאס דינאמי - לא יווצרו לנו קונפילקטים
